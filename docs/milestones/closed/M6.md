# M6 - Cutover

**Goal:** Promote the solo design set to `docs/` and archive the prior
design tree. After M6, the solo design set is the only design set.

**Dependencies:** M5 in the field for at least one cycle.

## Outcome

The cutover completed during earlier milestone work, not as a discrete
M6 epic. The solo design set was authored directly under `docs/design/`,
`docs/milestones/`, and `docs/operations/` - there was never a parallel
`docs/docsv2solo/` staging tree in this repo, and no pre-solo `DV2-`
design tree was ever committed. By the time M6 came up for planning, the
"promote and archive" work it described was already a no-op.

M6 is therefore closed as a documentation reconciliation: the exit gates
were verified against the live repo (audit, 2026-05-18) and all pass or
are moot. The four planned promotion epics (m6.01–m6.04) were dropped -
they would have produced only empty file moves.

- [x] No file under `docs/` references a `DV2-` ID. → Verified by grep. The only

  no actual `DV2-` identifiers exist.
- [x] The pre-solo tree exists only under `docs/archive/pre-solo/`. → Vacuously
  satisfied: no pre-solo design tree was ever committed, so no live old tree exists.
