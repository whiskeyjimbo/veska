# M4 — Wiki Mechanical Kinds

**Goal:** Two mechanical wiki kinds (`hot_zone`, `entry_points`)
rendered to `docs/veska/`, plus the `eng_get_context_pack` MCP tool
for agents. No LLM in the rendering path; pages are pure
functions of promoted graph state.

**Tracker:** epic `solov2-erp`.

**Dependencies:** M3.

## Exit gates

- [x] Both pages regenerate on promotion in < 5s for a typical commit.
- [x] Pages are deterministic given the same promoted state (test: same input → byte-identical output).
- [x] `eng_get_context_pack` returns under 50ms p95 for typical inputs.
- [x] No path in M4 calls the LLM generator. Verified by tests.

## Epics

### Epic m4.01 — Context pack

**Tracker:** `solov2-erp.4` (closed); depended on `solov2-erp.1` (git commit-history reader).

**DoD**
- [x] `eng_get_context_pack` MCP tool: input is `{symbol | task_id}`; output is a token-bounded JSON bundle of nodes, recent commits, open findings, and tasks relevant to the input.
- [x] Token budget configurable; default 8192.
- [x] Bundle structure documented — self-documenting in epic `solov2-erp.4`; cross-referenced from SOLO-09 (MCP surface). (See M4.5 finding for the SOLO-09 cross-reference task.)
- [x] Total tool count after M4: 27 (24 registered as of M3 close + `eng_get_context_pack` + `eng_get_hot_zone` + `eng_get_entry_points`).

**Sub-issues**

- `m4.01.2-budget` — token-counting helper.
