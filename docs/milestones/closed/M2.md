# M2 — Identity, Observability, Plugin Scaffolding

**Goal:** Plumb `actor_id` and `actor_kind` through every write;
land the single human-action gate; expand MCP to ~20 tools; declare the
plugin interfaces with one default impl each; turn on opt-in OTLP
and Prometheus; **ratify the vec0-to-HNSW pivot ADR against the
M0 measurement** (epic m2.06) so M3+ build on the pivoted index.

**Dependencies:** M1.

## Exit gates

- [x] Every state-changing MCP call writes `actor_id` and `actor_kind` on the affected row(s) and on the `audit.jsonl` line. — commit b9b486c; `Handler.Handle` carries `domain.Actor`; `Promoter.Promote` writes both columns; `ValidateAuditEntry` guards every write.
- [x] Human-action gate: `eng_close_finding` for `severity >= high` is refused when `actor_kind != 'human'`. Test covers refusal path for both `high` and `critical`. — commit 7ef962f; `CodeHumanRequired = -32001`; 4 test paths.
- [x] `metrics.enabled = false` ⇒ no HTTP listener bound. Verified by `veska doctor egress`. — commit baf362b; `StartHTTPListener` only called by caller; `CheckEgressObservability` enumerates.
- [x] OTLP exporter dispatches a span when `tracing.enabled = true` against a local collector. — commit baf362b; `NewTracerProvider(endpoint)` errors on empty; `parentbased_traceidratio` at 1.0.
- [x] Plugin interfaces stable; one impl per port; second-impl tests stubbed but skipped. — commit defaae3; `Tracker`, `VulnSource`, `Embedder`, `LLMGenerator`, `Notifier` + `ADR-TEMPLATE-second-impl.md`.
- [x] vec0-to-HNSW pivot ADR ratified: usearch (float16) confirmed as scale backend; sqlite-vec as default. M1 cleared 250k threshold. — commit 5a65f5e; `ADR-S0015-vec-pivot-dual-backend.md` (extends ADR-S0014).
