# M2 — Identity, Observability, Plugin Scaffolding

**Goal:** Plumb `actor_id` and `actor_kind` through every write;
land the single human-action gate; expand MCP to ~20 tools; declare the
plugin interfaces with one default impl each; turn on opt-in OTLP
and Prometheus; **ratify the vec0-to-HNSW pivot ADR against the
M0 measurement** (epic m2.06) so M3+ build on the pivoted index.

**Dependencies:** M1.

## Exit gates

- [ ] Every state-changing MCP call writes `actor_id` and `actor_kind` on the affected row(s) and on the `audit.jsonl` line.
- [ ] Human-action gate: `eng_close_finding` for `severity >= high` is refused when `actor_kind != 'human'`. Test covers refusal path for both `high` and `critical`.
- [ ] `metrics.enabled = false` ⇒ no HTTP listener bound. Verified by `veska doctor egress`.
- [ ] OTLP exporter dispatches a span when `tracing.enabled = true` against a local collector.
- [ ] Plugin interfaces stable; one impl per port; second-impl tests stubbed but skipped.
- [ ] vec0-to-HNSW pivot ADR (epic m2.06) ratified with M0-measured numbers, OR explicitly deferred to M3 because M0 cleared the 250 000-node threshold (SOLO-13 §3.3.1).

### Epic m2.06 — vec0-to-HNSW pivot ADR

**Tracker:** `veska:m2.06-vec-pivot-adr`

ADR-S0001's "staged adoption" preamble names this as the
canonical home for the HNSW pivot ADR. It is **mandatory at M2**
when the M0 measurement (epic m0.01) shows the brute-force vec0
ceiling falls below 250 000 embedded nodes (per SOLO-13
§3.3.1). When the M0 measurement clears 250 000, the epic ships
as a planned-follow-up ADR rather than a milestone-blocker;
either way M3+ build on the pivoted index, not vec0.

**DoD**
- [ ] ADR records the chosen backing (lancedb embedded vs. hnswlib via cgo) with measured numbers.
- [ ] ADR specifies the recall floor against a fixed eval set.
- [ ] ADR documents how the choice preserves the SOLO-08 §9 single-tarball backup property (or names the regression).
- [ ] `vector_index` port (SOLO-07 §4) gains the pivoted impl; the vec0 impl stays under a feature flag for the M2 transition window only.

**Sub-issues**
- `m2.06.1-bench` — ANN recall + latency bench against the M0 corpus on the reference laptop.
- `m2.06.2-impl` — pivoted impl behind the same `VectorIndex` port.
- `m2.06.3-backup-property` — verify `veska backup create` / `restore` still produces and consumes a single-tarball after the swap.
- `m2.06.4-adr` — write the ADR with measurements in hand.
