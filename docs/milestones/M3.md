# M3 ‚Äî Pipelines and Embedder

**Goal:** Promotion pipeline runs structural checks synchronously; an
async embedder worker keeps `node_embedding_refs` drained; vec0
search is the live path for `semantic_search`; auto-link
suggestions land; revalidation sweeps invalidate stale findings on
content drift.

**Dependencies:** M1, M2.

## Exit gates

- [ ] Embedder throughput on local CPU Ollama: measured at 5‚Äì20 emb/s on the reference laptop.
- [ ] `semantic_search` recall ‚â• 0.85 on the spike's eval set at 1M nodes (resolves OQ-S003) ‚Äî or an ADR for an alternative vector index.
- [ ] Auto-link false-positive rate measured on a fixture and surfaced in the milestone close report.
- [ ] Revalidation completes in < 60s for a 10k-edge commit.
- [ ] All four post-promotion queue `work_kind` lanes drain to `state='done'` under a fuzz workload.

## Epics

### Epic m3.01 ‚Äî Promotion pipeline

**Tracker:** `veska:m3.01-promotion-pipeline`

**DoD**
- [ ] Structural checks run inside the promotion transaction or immediately after, before hook return.
- [ ] Findings emitted with `source_layer='structural'`.
- [ ] Check set: parse-failure, dead-code (no inbound edges), contract-drift (signature change vs. prior promotion).
- [ ] Per-check timing surfaced in `veska_seal_latency_seconds`.

**Sub-issues**
- `m3.01.1-check-runner` ‚Äî synchronous check pipeline.
- `m3.01.2-parse-failure` ‚Äî surface tree-sitter errors as findings.
- `m3.01.3-dead-code` ‚Äî inbound-edge sweep.
- `m3.01.4-contract-drift` ‚Äî signature comparison.

### Epic m3.02 ‚Äî Embedder worker

**Tracker:** `veska:m3.02-embedder-worker`

**DoD**
- [ ] Goroutine drains `node_embedding_refs` where `state='pending'`.
- [ ] Throttled by `embedder.rate_per_sec` config; default 10.
- [ ] Backpressure via `veska_embed_queue_depth`.
- [ ] Failed batches retry up to 3; `state='failed'` after that, surfaced in `veska doctor`.

**Sub-issues**
- `m3.02.1-batch-loop` ‚Äî pull pending; batch by 32; embed.
- `m3.02.2-rate-limit` ‚Äî token bucket on requests per second.
- `m3.02.3-error-handling` ‚Äî retry policy; failed-state surfacing.
- `m3.02.4-content-addressed` ‚Äî dedupe by `content_hash` before calling Ollama.

### Epic m3.04 ‚Äî Auto-link

**Tracker:** `veska:m3.04-auto-link`

**DoD**
- [ ] post-promotion queue `work_kind='auto_link'` proposes `Edge` rows with `confidence='unresolved'` from embedding similarity.
- [ ] Threshold configurable; default tuned on the eval set.
- [ ] Surfaces as findings (`source_layer='semantic'`), not silently inserted edges.
- [ ] No auto-merge; the developer accepts or suppresses.
- [ ] Accept-flow uses existing `eng_close_finding` with
  `reason="accept"`; the close handler promotes the linked
  unresolved edge to `confidence='definite'`. No new MCP tool.

**Sub-issues**
- `m3.04.1-similarity` ‚Äî top-k similarity per node.
- `m3.04.2-finding-shape` ‚Äî auto-link finding kind.
- `m3.04.3-accept-flow` ‚Äî `eng_close_finding[reason="accept"]` promotes the linked edge.
- `m3.04.4-fp-measurement` ‚Äî false-positive rate on fixture.

### Epic m3.05 ‚Äî Revalidation

**Tracker:** `veska:m3.05-revalidate`

**DoD**
- [ ] post-promotion queue `work_kind='revalidate'` re-checks open findings whose anchor `content_hash` changed.
- [ ] Findings transition to `closed` with `closed_reason='revalidated_obsolete'` (rule no longer fires) or `closed_reason='superseded_by_revalidation'` (rule fires on new content; new finding linked back). State stays in `{open, closed}` ‚Äî no `superseded` state. (SOLO-04 ¬ß8.1.)
- [ ] Sweep runs after promotion; bounded to the symbols touched by the promotion.

**Sub-issues**
- `m3.05.1-anchor-tracking` ‚Äî finding ‚Üî node hash linkage.
- `m3.05.2-sweep` ‚Äî revalidate goroutine.
- `m3.05.3-supersede` ‚Äî chain old ‚Üí new finding.
- `m3.05.4-bench` ‚Äî 10k-edge commit revalidation timing.
