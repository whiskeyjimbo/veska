# M4 — Wiki Mechanical Kinds

**Goal:** Two mechanical wiki kinds (`hot_zone`, `entry_points`)
rendered to `docs/veska/`, plus the `eng_get_context_pack` MCP tool
for agents. No LLM in the rendering path; pages are pure
functions of promoted graph state.

**Tracker:** epic `solov2-erp`.

**Dependencies:** M3.

## Exit gates

- [ ] Both pages regenerate on promotion in < 5s for a typical commit.
- [ ] Pages are deterministic given the same promoted state (test: same input → byte-identical output).
- [ ] `eng_context_pack` returns under 50ms p95 for typical inputs.
- [ ] No path in M4 calls the LLM generator. Verified by tests.

## Epics

### Epic m4.01 — Context pack

**Tracker:** `veska:m4.01-context-pack`

**DoD**
- [ ] `eng_get_context_pack` MCP tool: input is `{symbol | task_id}`; output is a token-bounded JSON bundle of nodes, recent commits, open findings, and tasks relevant to the input.
- [ ] Token budget configurable; default 8192.
- [ ] Bundle structure documented in SOLO-09.
- [ ] Total tool count after M4: 27 (24 registered as of M3 close + `eng_get_context_pack` + `eng_get_hot_zone` + `eng_get_entry_points`).

**Sub-issues**
- `m4.01.1-handler` — tool implementation.
- `m4.01.2-budget` — token-counting helper.
- `m4.01.3-symbol-mode` — pack-around-symbol.
- `m4.01.4-task-mode` — pack-around-task.

**Tracker:** `veska:m4.02-hot-zone`
