# Fat-file LOC ratchet inventory (solov2-0omh.3)
#
# Each record is `<path> <maxLOC>`: the recorded ceiling is the file's TOTAL
# line count (wc -l) at the time it was inventoried. `make fatfile-ratchet`
# fails if any listed file GROWS past its ceiling. This complements
# `make lint-size` (per-FUNCTION limits on CHANGED code) by ratcheting the
# total size of pre-existing fat files DOWN over time — grandfathering can no
# longer hide them forever.
#
# THRESHOLD: seeded with every first-party non-test .go file under cmd/ and
# internal/ that is >= 500 LOC. (Excludes _test.go, tools/, loadtest corpora,
# and vendored/embedded assets.)
#
# HOW TO UPDATE AN ENTRY:
#   - After thinning a file (WS-2 tasks .4-.7), run `wc -l <path>` and set the
#     record to the NEW (smaller) value. The ratchet only ratchets DOWN: never
#     raise a number to make the gate pass.
#   - When a file drops below ~500 LOC you may delete its line entirely.
#   - If a file is renamed/removed, update or delete its line (a stale entry
#     for a missing file is itself a failure).
#   - Rare exception to "only ratchet down": a +1 here on doctor.go/search.go
#     (solov2-0omh.4) reflects a single new cross-package import line added
#     when shared CLI helpers moved into internal/cli/repocmd — a structural
#     improvement, not logic growth. Both files are thinned to ~200 LOC by
#     .6/.5, which will delete these lines outright.
#
# Blank lines and `#` comments are ignored.
